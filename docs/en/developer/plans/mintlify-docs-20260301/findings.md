{/* Replace legacy docs wording with framework-agnostic phrasing. docs/en/developer/plans/mintlify-docs-20260301/task_plan.md mintlify-docs-20260301 */}
- The `docs/` directory previously contained a legacy docs site configuration; those files were removed during this Mintlify refactor.
 - The `docs/en/` tree includes `api-reference`, `developer`, `user-docs`, and `change-log` sections.
 - API reference pages previously used legacy OpenAPI MDX components that were replaced with Mintlify-friendly endpoint maps.
{/* Capture Mintlify documentation mechanics for navigation and OpenAPI integration. docs/en/developer/plans/mintlify-docs-20260301/task_plan.md mintlify-docs-20260301 */}
- Mintlify uses `docs.json` to define navigation; pages not in `docs.json` are hidden from sidebar but can still be accessed by direct links or search.
- Mintlify navigation supports tabs, groups, and pages, and folder names like `api-reference` are allowed (top-level `api` is reserved).
- Mintlify supports OpenAPI-based docs by referencing an OpenAPI file or URL in `docs.json` and generating endpoint pages from navigation entries.
- Mintlify callouts are rendered via MDX components such as `<Note>`, `<Tip>`, `<Warning>`, `<Info>`, `<Check>`, and `<Danger>`.
{/* Track Mintlify component and navigation constraints to guide refactor decisions. docs/en/developer/plans/mintlify-docs-20260301/task_plan.md mintlify-docs-20260301 */}
- Mintlify navigation is defined inside `docs.json` with root elements such as tabs, groups, or pages; nested groups/pages are allowed and folders map to URL paths by directory structure.
- Mintlify OpenAPI pages can be auto-generated by attaching an `openapi` field to navigation entries that points to a local file path or a hosted URL.
- Mintlify callouts can also be rendered via a `<Callout>` component for more customized callout blocks.
{/* Capture Mintlify navigation constraints and OpenAPI auto-generation specifics. docs/en/developer/plans/mintlify-docs-20260301/task_plan.md mintlify-docs-20260301 */}
- Mintlify navigation requires a root-level navigation element (tabs, groups, or pages) and allows nested structures under that root.
- Mintlify navigation does not allow using a top-level `api` folder name, but folders like `api-reference` are allowed.
- Mintlify auto-generates OpenAPI endpoint pages by adding an `openapi` field to a navigation element; the field can point to a local file path or a URL.
{/* Capture OpenAPI navigation behavior for auto-generated pages. docs/en/developer/plans/mintlify-docs-20260301/task_plan.md mintlify-docs-20260301 */}
- Mintlify navigation can attach `openapi` to a group/tab/anchor; if `openapi` is set and no pages are listed, Mintlify generates pages for all endpoints in the spec, while listing pages explicitly limits which endpoints appear.
{/* Capture Mintlify callout component usage and variants. docs/en/developer/plans/mintlify-docs-20260301/task_plan.md mintlify-docs-20260301 */}
- Mintlify supports a `Callout` MDX component with a `type` prop (e.g., `note`, `tip`, `warning`, `danger`, `check`) and optional `title` for styled callout blocks.
{/* Capture Mintlify card components for navigation-style layouts. docs/en/developer/plans/mintlify-docs-20260301/task_plan.md mintlify-docs-20260301 */}
- Mintlify supports `Card` and `CardGroup` components for grid-style navigation, and `Card` can accept a title, icon, and href for linkable tiles.
{/* Capture Mintlify navigation root requirements and OpenAPI inheritance details. docs/en/developer/plans/mintlify-docs-20260301/task_plan.md mintlify-docs-20260301 */}
- Mintlify requires choosing a primary navigation pattern at the root of `navigation` in `docs.json`, and nested navigation elements can be combined under it.
- When `openapi` is set on a navigation element, child elements inherit the spec unless they define their own, and listing explicit endpoints in `pages` limits which endpoints are generated.
{/* Record a public example of docs.json in a Mintlify starter repo. docs/en/developer/plans/mintlify-docs-20260301/task_plan.md mintlify-docs-20260301 */}
- The Mintlify starter repository includes a `docs.json` config file at repo root, confirming the expected config filename and structure.
